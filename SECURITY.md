# Seguridad

Gracias por tomarte el tiempo de revisar la seguridad de Pinyin Tones.

## Cómo reportar una vulnerabilidad

Si encontrás una posible vulnerabilidad, evitá publicarla en un issue abierto. Como mantenedor único, prefiero que el reporte llegue con el mayor nivel de detalle posible, pero sin exponer información sensible innecesaria.

Incluí, si podés:

- descripción breve del problema;
- versión de la aplicación o commit afectado;
- sistema operativo;
- pasos mínimos para reproducirlo;
- impacto esperado;
- cualquier captura, log o prueba de concepto relevante.

## Qué esperar

Voy a revisar el reporte lo antes posible y responder con una confirmación de recepción o con preguntas para reproducir el problema.

## Buenas prácticas

- No compartas exploits completos en público si la vulnerabilidad todavía no fue corregida.
- Si el problema afecta a una release publicada, mencioná qué versión está impactada.
- Si el hallazgo está relacionado con configuraciones, indicá los valores usados.

## Alcance

Este documento cubre vulnerabilidades en el código, empaquetado y comportamiento de la aplicación `pinyin-tones`.

Si el reporte no corresponde a seguridad sino a un error funcional común, usá un issue normal o la plantilla de bug report.
